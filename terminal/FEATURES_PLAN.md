## 待实现功能 ❌

### 优先级 P0 - 核心体验

#### 1. WebSocket 实时连接
**现状**: 终端前端目前使用离线模式，没有与后端 WebSocket 连接。
**需求**: 
- 连接后端 WebSocket 服务
- 接收实时游戏状态更新
- 发送玩家操作（ready/unready/action/chat）

**涉及文件**: 
- 新建 `websocket_client.py`
- 修改 `poker_terminal.py`

#### 2. 聊天系统
**现状**: 无聊天功能。
**需求**:
- 显示聊天消息区域
- 支持发送消息
- 区分系统消息和玩家消息

**涉及文件**: 
- 修改 `ui/display.py` - 添加聊天区域显示
- 修改 `poker_terminal.py` - 添加聊天输入处理

#### 3. 补充筹码（Rebuy）
**现状**: 玩家筹码为 0 时无法继续游戏。
**需求**:
- 检测玩家筹码为 0 时提示补充
- 提供补充筹码界面
- 调用 API 补充筹码

**涉及文件**: 
- 修改 `lobby.py` - 添加 rebuy API 调用
- 修改 `poker_terminal.py` - 添加补充筹码提示

#### 4. 切换座位
**现状**: 无法切换座位。
**需求**:
- 提供切换座位选项
- 调用 API 切换座位

**涉及文件**: 
- 修改 `api_client.py` - 添加 switch_seat 方法
- 修改 `poker_terminal.py` - 添加座位选择界面

## 技术要点

### WebSocket 连接
- 使用 `websockets` 库进行 WebSocket 通信
- JWT Token 通过 URL 参数传递
- 支持自动重连

### 聊天系统
- 使用 Rich 的日志区域或单独面板
- 支持历史消息滚动
- 系统消息特殊样式

### 倒计时
- 使用 threading.Timer 或 asyncio
- 服务器剩余时间同步
