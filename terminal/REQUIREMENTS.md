# 德州扑克终端前端 - 需求规格说明书

## 一、项目概述

基于现有 Web 前端功能，实现一个功能完整的终端版本德州扑克游戏前端。

## 二、核心需求

### 2.1 游戏流程功能

#### 2.1.1 游戏前阶段 (Pre-game)
- [ ] 设置玩家数量 (2-10 人)
- [ ] 设置起始筹码
- [ ] 玩家准备/取消准备状态
- [ ] 房主开始游戏
- [ ] 筹码不足时补充筹码

#### 2.1.2 游戏进行阶段 (In-game)
- [ ] 四轮下注流程（翻牌前、翻牌、转牌、河牌）
- [ ] 玩家操作：弃牌、过牌、跟注、加注、全押
- [ ] 盲注自动扣除
- [ ] 庄家位置轮换
- [ ] 手牌发放和显示

#### 2.1.3 结算阶段
- [ ] 摊牌比牌
- [ ] 底池分配
- [ ] 获胜者显示
- [ ] 牌型展示
- [ ] 净筹码计算

## 五、技术实现要点

### 5.1 状态刷新机制
使用 `rich.live.Live` 实现实时刷新：
```python
from rich.live import Live

with Live(refresh_per_second=4) as live:
    while game_active:
        live.update(render_game_state())
        time.sleep(0.1)
```

### 5.2 布局方案
使用 `rich.layout.Layout` 分割屏幕：
- 顶部：游戏信息（阶段、公共牌、底池）
- 中央：牌桌和玩家
- 底部：操作面板/等待提示

### 5.3 输入处理
使用 `rich.prompt` 系列：
- `Prompt.ask` - 文本选择
- `IntPrompt.ask` - 数字输入
- `Confirm.ask` - 确认对话框
