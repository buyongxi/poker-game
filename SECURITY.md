# 安全政策

我们认真对待安全问题。如果你发现了漏洞或潜在安全风险，请按以下方式反馈。

## 漏洞披露

- 请通过 [Issues](https://github.com/your-username/poker-game/issues) 创建讨论（或直接发起 PR）
- 描述漏洞现象、影响范围、复现步骤与建议修复方向
- 若涉及敏感信息泄露风险，建议不要在 Issue 中贴出完整密钥/令牌/用户隐私数据

## 安全建议

### 部署生产环境时

1. **SECRET_KEY**
   - 务必使用强随机字符串（至少 32 字符）
   - 不要使用示例或默认值
   ```bash
   # 生成随机密钥示例
   python -c "import secrets; print(secrets.token_urlsafe(32))"
   ```

2. **管理员密码**
   - 使用强密码（大小写字母 + 数字 + 符号，至少 12 字符）
   - 定期更换密码
   - 不要将密码提交到版本控制

3. **环境变量**
   - 确保 `.env` 文件不被提交到版本控制（已在 `.gitignore` 中排除）
   - 生产环境使用独立的环境变量管理

4. **数据库**
   - 生产环境建议使用 PostgreSQL 等更强大的数据库
   - 定期备份数据库文件

5. **CORS 配置**
   - 根据实际部署域名配置 `CORS_ORIGINS`
   - 不要在生产环境使用通配符 `*`

6. **HTTPS**
   - 生产环境务必使用 HTTPS
   - WebSocket 连接使用 `wss://` 协议
